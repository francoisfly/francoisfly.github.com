利用URL Protocol调用本地EXE程序

利用URL Protocol调用本地EXE程序

六月 03, 2019
  • URL Protocol
  • regedit

在网上经常可以看到点击一个按钮或者一个连接直接打开QQ聊天框。这种方式都是通过URL Protocol方式实现的。URL Protocol它可以兼容各种浏览器。

第一步：先注册URL PROTOCOL

方法一：新建一个扩展名为：reg的文件。

第一行：注册表工具的版本信息。

第二行：在注册表【HKEY_CLASSES_ROOT】节点下创建名为【openlayer】的节点。其中【openlayer】的名称就对应着自定义URL Protocol的名称，在web中调用的时候需要这个名称。

第三行：x:\\x.exe 表示要打开程序的路径。

第四行：协议的名称，可以是任意字符串，后面不会用到。

第六行：x:\\x.exe也是对应的程序路径，后面的1照抄。

第十行：x:\\x.exe也是对应的程序路径，其中%1表示到参数，参数可以在你的程序openlayer.exe中解析得到。

保存之后双击该文件即可直接导入到注册表中。按下win + r 打开 输入regedit 即可打开注册表。

在【HKEY_CLASSES_ROOT】节点下可以找到该【openlayer】的节点名称。

方法二：通过C#语句进行URL Protocol注册

第二步：应用

传递参数

传递参数：写注册表的时候，最后路径要加上参数[HKEY_CLASSES_ROOT\openlayer\shell\open\command]

@="\"C:\\WINDOWS\\NOTEPAD.EXE\" \"%1\""

1)、以C#为例，在程序的主入口加入参数。

2）、在写链接的时候自定义参数，程序会把整个链接都获取到，需要自己在进行解析。

程序会输出一下结果：openlayer://msg
